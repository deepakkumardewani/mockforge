## Design Context

### Users
Developers who need mock/fake data APIs for prototyping and testing. They use this in a terminal-adjacent context — building apps, running tests, evaluating API shapes. They are technically skilled and value speed, clarity, and zero friction.

### Brand Personality
**Three words**: precise · inevitable · forged

MockForge should feel like a high-precision tool — not a startup landing page. Quiet confidence. Every detail deliberate. The name implies craft and heat (a forge), not fluff.

### Aesthetic Direction
- **Theme**: Dark default (`oklch(0.10 0.006 65)` base), light supported via existing ThemeToggle
- **Accent**: Warm amber `oklch(0.78 0.16 65)` dark / `oklch(0.58 0.18 65)` light — the heat of the forge, distinctive from the indigo-heavy dev tool landscape
- **Typography**: Bricolage Grotesque (display/headings, variable weight), system sans for body
- **Visual tone**: Asymmetric layouts, editorial hierarchy, oversized text as visual anchors — not centered-everything card grids
- **Anti-references**: Generic AI slop — gradient text, icon-in-rounded-squares, same-sized card grids, radial glow blobs, centered blob layouts

### Design Principles
1. **Asymmetry over symmetry** — left-aligned text, two-column heroes, break the grid with intent
2. **Typography does the heavy lifting** — oversized labels, tracked uppercase labels, clip-path reveals
3. **Amber is the only accent** — use it sparingly: labels, active states, the single rule line, cursor blink. Never gradient it.
4. **Motion has two layers** — choreographed entrance (GSAP clip-path stagger) + ambient CSS background (no JS)
5. **No AI tells** — no gradient text, no border-left stripes, no glassmorphism, no icon-in-rounded-square, no radial glow blobs as decor
